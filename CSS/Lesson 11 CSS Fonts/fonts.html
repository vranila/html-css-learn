CSS Fonts — Practical Guide

Choose readable, performant fonts. Use fallbacks. Respect accessibility and brand.

Why Font Selection Matters

The right font strengthens brand identity and affects readability, perception, and trust. On screens, prefer sans-serif for UI; use serif for editorial tone when appropriate.

Font-Family: Stacks & Fallbacks

Serif stack: font-family: "Merriweather", Georgia, "Times New Roman", serif;

Sans-serif stack: font-family: "Inter", system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

Monospace stack: font-family: "Courier New", Courier, "Lucida Console", monospace;

Tip: Put the preferred web font first; finish with a generic family (serif, sans-serif, monospace).

Weight, Style, Size, Line-height

Weight & style

300 — Light

400 — Regular

600 — Semi-bold

800 — Bold

Size & line-height (readability)

16px / line-height: 1.5 is standard for body copy on desktop.

Clamp for responsive sizing: font-size: clamp(1rem, 1.2vw + 0.6rem, 1.125rem);

body {
  font-family: "Inter", system-ui, -apple-system, "Segoe UI", Roboto, sans-serif;
  font-size: 16px;
  line-height: 1.5;
  color: #111827;
}

Using Web Fonts

@font-face (self-hosted example)

@font-face {
  font-family: "BrandSans";
  src: url("fonts/BrandSans-Regular.woff2") format("woff2"),
       url("fonts/BrandSans-Regular.woff") format("woff");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

Google Fonts (recommended workflow)

Import via <link> (shown in the <head>) and use the family name directly in CSS.

/* example: Google Font 'Inter' loaded in the head */
h1 { font-family: "Inter", system-ui, sans-serif; }
Use font-display: swap to avoid invisible text during font loading and improve perceived performance.

Accessibility & Performance

  • Contrast: Ensure text color passes WCAG contrast (≥4.5:1 for normal text).
  • Fallbacks: Include system fallbacks to avoid layout shift.
  • Performance: Limit web font variants (weights/styles) to reduce payload.
  • Legibility: Prefer 16px+ base, 1.4–1.6 line-height for body text.
/* performance tip */
@font-face { font-display: swap; } /* avoid FOIT (flash of invisible text) */

Practical Patterns & Tips

  1. UI headings: Use system stack for instant rendering — e.g. font-family: Inter, system-ui, sans-serif;.
  2. Editorial body: Use a readable serif or robust sans with moderate line-height.
  3. Icon fonts: Prefer SVG or icon fonts with CSS fallback.
  4. Avoid: too many custom web fonts and dramatic uppercase on long copy.
/* Responsive headline example */
h1 {
  font-family: "Merriweather", Georgia, serif;
  font-size: clamp(1.75rem, 2.5vw + 0.5rem, 3rem);
  line-height: 1.1;
  font-weight: 700;
}
