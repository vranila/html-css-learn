CSS Errors

Errors in CSS can cause styles to behave unpredictably or not apply at all. While most browsers ignore invalid rules, missing punctuation or braces can break large parts of your stylesheet.

1. Missing Semicolons

Each property must end with a semicolon (;). Missing one can stop the next property from being read correctly.


.bad {
  color: red
  background-color: yellow;
}


.good {
  color: red;
  background-color: yellow;
}

2. Invalid Property Names

Misspelled or non-existent properties are ignored by browsers.


.bad {
  colr: blue;        /* Invalid property name */
  font-size: 16px;
}


.good {
  color: blue;
  font-size: 16px;
}

3. Invalid Values

If you use a valid property but an invalid value, that rule won’t apply.


.bad {
  width: -100px;   /* Negative width is invalid */
  color: green;
}


.good {
  width: 100px;
  color: green;
}

4. Unclosed Braces

Forgetting to close a curly brace (}) can invalidate all following CSS rules.


.bad {
  padding: 20px;
  margin: 10px;


.good {
  padding: 20px;
  margin: 10px;
}

5. Extra Colons or Braces

Accidentally typing extra colons or braces can break the syntax.


.bad {
  color:: blue;
}


.good {
  color: blue;
}

Tips to Avoid CSS Errors

  • Use a code editor with syntax highlighting (e.g., VS Code).
  • Validate CSS using an online CSS validator or linter.
  • Write CSS in small sections and test frequently in a browser.
  • Use proper indentation and comments for clarity.
